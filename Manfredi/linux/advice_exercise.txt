Regarding exercise 1, we had already corrected it in class if you were present, but everything is fine anyway. The only small improvement could be in the first grep: instead of searching for the string "sp", you could look for the character ">" (which is always present in FASTA files), making it a bit more generic. However, this is not essential.

For exercise 2, everything is fine except for the last command. If the task is to write the amino acid sequence of the protein to a file, performing a sort breaks the sequence information. Therefore, it would be better to avoid it and print the sequences in the order they appear in the file. Including the residue number was not required, but it makes sense as useful information to keep, so the cut command is fine.

For exercise 3, just two small things. The requirement was to print the values in descending order, so the -n flag for numerical sorting is correct, but another flag is needed to reverse the order. As a second note—just a stylistic suggestion, since the result doesn’t change and your solution is correct—I would personally use tail before cut. Logically, you first want to remove the header and then extract the columns of interest from what remains. Again, this makes no difference in practice, but in this way you avoid cutting random characters from the first 28 lines. You can imagine that in an intermediate output, the result of cut might be harder to interpret.

For exercise 4, please re-read the assignment! What you found is the number of sequences marked with 0 and 1, but the actual request was to count how many residues belong to such sequences.
