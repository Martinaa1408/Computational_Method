

# 🔹 **A. Quando leggi un file riga per riga**

### ✔ 1. Sempre, sempre, sempre `strip()`

```python
line = line.strip()
```

### ❗ Perché è obbligatorio

* Senza strip → newline rimane (`"\n"`), `split()` produce miracoli strani, confronti falliscono.

### ✔ 2. Controlla IMMEDIATAMENTE le righe vuote

```python
if line == "":
    continue
```

**Errore classico da evitare:**
usare `parts = line.split()` *prima* di controllare se è vuota → `parts` diventa `[]` e poi fai `parts[0]` → CRASH.

---

# 🔹 **B. Uso corretto di `split()`**

### ✔ 3. Usa SEMPRE `split()` senza argomenti

È lo standard dell’esame.

```python
parts = line.split()
```

### Cosa garantisce:

* Niente parole vuote
* Gestisce spazi multipli
* Gestisce tab e spazi insieme

### ❌ Mai usare `split(" ")`

→ crea elementi vuoti se ci sono 2 spazi consecutivi.

---

# 🔹 **C. Prima di accedere a `parts[x]`, fai sempre un controllo**

### ✔ 4. Regola tecnica obbligatoria:

```python
if len(parts) < N:
    continue
```

Dove **N** è l’indice che vuoi usare + 1.

Esempio:
Se vuoi usare `parts[2]`, fai:

```python
if len(parts) < 3:
    continue
```

### ❗ Perché è fondamentale:

Lanese mette SEMPRE almeno un caso “sporco”.

---

# 🔹 **D. Conversione tipi: quando serve `str()`**

### ✔ 5. Se devi concatenare una stringa a un numero → usa `str()`

```python
out = key + " " + str(value)
```

### ✔ 6. Se fai `join`, ogni elemento deve essere stringa

```python
values = [str(v) for v in values]
line = "\t".join(values)
```

### ❌ Errore tipico:

```python
"\t".join([word, 34])  # CRASH
```

---

# 🔹 **E. Quando usare `line` e quando `parts`**

### ✔ 7. Usa `line` quando:

* vuoi controllare se inizia con un certo carattere (`line.startswith(">")`)
* vuoi tenere l’intera riga
* vuoi saltare commenti

### ✔ 8. Usa `parts` quando:

* devi estrarre *campi*
* devi contare *parole*
* devi prelevare colonne (`parts[0]`, `parts[1]`, ecc.)

### ❗ Regola di ferro:

> Se devi usare indici → lavora con `parts`, *mai* con `line`.

---

# 🔹 **F. Uso corretto di `continue`, `break`, `return`**

### ✔ 9. `continue` serve SOLO per saltare l’iterazione corrente

```python
if bad:
    continue
```

### ✔ 10. `break` lo usi solo per uscire da *un ciclo*

(mai dalla funzione)

### ✔ 11. `return` chiude TUTTA la funzione

→ nei file-parser è quasi sempre sbagliato
→ lo usi solo se all’esame ti viene chiesto di terminare subito riportando un valore.

### ❗Errore comune:

Usare `return` dentro un ciclo pensando di saltare “solo quella riga”.

---

# 🔹 **G. Dizionari: regole meccaniche**

### ✔ 12. Inserire valori nel dizionario senza errori:

```python
if key not in d:
    d[key] = 0
d[key] += 1
```

### ✔ 13. Forma compatta corretta:

```python
d[key] = d.get(key, 0) + 1
```

### ❗ Mai fare:

```python
d[key] += 1   # se la key non c’è → CRASH
```

---

# 🔹 **H. Liste: aggiungere elementi nel modo corretto**

### ✔ 14. Usa `append()` sempre, mai `+` con liste:

```python
words.append(word)
```

Mai:

```python
words = words + [word]  # lento e pericoloso
```

### ✔ 15. Per resettare una lista dentro un ciclo:

```python
current = []
```

non:

```python
current = current.clear()  # sbagliato: clear() restituisce None
```

---

# 🔹 **I. Uso perfetto di `join`**

### ✔ 16. `join` funziona solo su liste di stringhe:

Corretta:

```python
"\t".join([str(a), b, c])
```

### ✔ 17. Non usare `join` su liste di liste o liste miste.

---

# 🔹 **L. Ordine di operazioni giusto in tutti i compiti**

### ✔ 18. La pipeline corretta in TUTTI gli esercizi di parsing è:

1. `line.strip()`
2. salta se vuota
3. `split()`
4. controlla lunghezza
5. prendi valori
6. converti tipi
7. aggiorna dizionari/liste
8. produci output

**Mai invertire 3 e 4**, mai prendere indici prima di verificare la lunghezza.

---

# 🔹 **M. Evitare problemi con parole maiuscole/minuscole**

### ✔ 19. Se devi confrontare stringhe:

```python
word = word.lower()
```

oppure

```python
line = line.upper()
```

Lanese ama mettere varianti del tipo "CAT / cat / Cat".

---

# 🔹 **N. Errori tipici che devi assolutamente evitare**

### ❌ Usare `return` nel posto sbagliato

### ❌ Fare `parts = line.split()` PRIMA di eliminare righe vuote

### ❌ Accedere a `parts[1]` senza controlli

### ❌ Passare numeri a `join`

### ❌ Dimenticare `strip()`

### ❌ Cambiare nomi delle variabili richieste dal testo

### ❌ Usare concatenazioni confuse (`out = out + word`) anziché `append` o `join`

### ❌ Usare `split(" ")` invece di `split()`
